# AGENTS.md

## 適用範囲

この指示は `aegis-arc/` 配下の作業に適用する。

## プロジェクト

- Astro製の静的サイト。
- デザインは「学院・機関・近未来」を核とし、濃紺・白・シアンを基調とする。
- ギャラリーページは `src/pages/gallery.astro` で表示する。
- ギャラリーデータは `src/data/gallery.yml` で管理する。
- ギャラリー画像は `public/assets/img/gallery/` 配下で管理する。

## ギャラリー更新ルール

ユーザーがギャラリー画像の追加や更新を依頼した場合は、次のルールに従う。

1. `public/assets/img/gallery/` の画像一覧と `src/data/gallery.yml` の `src` を照合する。
2. YAMLに未登録の画像だけを追加する。
3. 画像は原則としてキャラクターごとのディレクトリに配置する。
4. `tags` に少なくとも1つの正式なキャラクター名を設定する。
5. 複数のキャラクターが写る画像は `tags` に複数指定する。
6. `title` は追加しない。ギャラリーにタイトルの概念はない。
7. `description` は任意項目とし、ユーザーが文言を指定した場合だけ設定する。内容を推測して追加しない。
8. `position` は人物の顔がサムネイル内に収まるように設定する。指定不要な場合は省略できる。
9. フィルター一覧は `tags` から自動生成されるため、別の `filters` 設定は追加しない。
10. `id` は `AA-G-0001` 形式のギャラリー固有IDとし、既存項目と重複しない次の番号を設定する。並べ替え時も既存IDは変更しない。

## YAML形式

```yml
items:
  - id: AA-G-0001
    src: /assets/img/gallery/ritsu/example.jpeg
    tags: [千景 律]
    description: ユーザーが指定した場合のみ記載
    position: 50% 25%
```

`description` と `position` は任意。省略時はキー自体を記載しない。

- `tokitowa/towa` → `巡 永`

### 双子キャラクター「巡 刻・巡 永」の配置ルール

- `tokitowa/` を双子共通の親ディレクトリとする。
- 巡 刻のみが写る画像は `tokitowa/toki/` に配置し、`tags: [巡 刻]` で登録する。
- 巡 永のみが写る画像は `tokitowa/towa/` に配置し、`tags: [巡 永]` で登録する。
- 二人が写る構図の画像はサブフォルダーに分けず `tokitowa/` 直下に配置し、`tags: [巡 刻, 巡 永]` で登録する。

新しいディレクトリやキャラクターを追加する際は、既存のプロット表記と照合する。正式名が判断できない場合はユーザーに確認する。

## 更新後の確認

- `src` の重複がないこと。
- `id` が全項目にあり、重複していないこと。
- YAMLの全 `src` に対応する画像ファイルが存在すること。
- タグ絞り込みが正常に動作すること。
- 複数タグの画像が各タグで表示されること。
- `description` 未設定時に説明欄が表示されないこと。
- `npm run build` が成功すること。

## 変更時の注意

- 既存の濃紺・白・シアンを基調としたデザインを維持する。
- 画像の拡大表示、前後移動、Escキー操作、キーボードフォーカスを壊さない。
- 静的ビルドとCloudflare Pagesでの公開を前提にする。
